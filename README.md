# ESP32 UART to UDP Bridge

![Логотип](images/img0.png)

Проект превращает ESP32 в мост между UART и UDP. Устройство принимает данные по UART, отправляет их по UDP на указанный IP-адрес или в broadcast, а при экранной сборке может показывать состояние на OLED-дисплее.

Сейчас проект поддерживает две аппаратные конфигурации:

- `ESP32-C3`
- `ESP32-S2 Mini`

## Возможности

- Передача данных из UART в UDP на порт `8888`
- Wi-Fi в режиме `STA` с автоматическим переходом в `AP`, если сеть недоступна
- Настройка параметров через веб-интерфейс `SettingsGyver`
- Опциональный OLED-дисплей `SSD1306 128x64`
- Режим `broadcast`
- Режим `echo` для отладки
- Внешний экран по UDP на порт `82` для экранной сборки
- Настройка мощности Wi-Fi
- Настройка яркости локального OLED через числовое поле `0..255`
- Линия сброса внешнего устройства
- Безэкранный вариант сборки через compile-time флаг

## Поддерживаемые платы

### ESP32-C3

- Environment в `PlatformIO`: `esp32-c3-devkitm-1`
- Дисплей: `SSD1306 128x64` по `SPI`
- UART:
  - `TX = GPIO21`
  - `RX = GPIO20`
- Линия сброса внешнего устройства: `GPIO9`

Подключение дисплея:

```text
ESP32-C3     OLED SSD1306
GPIO 6   ->  MOSI
GPIO 4   ->  CLK
GPIO 3   ->  DC
GPIO 1   ->  CS
GPIO 2   ->  RESET
```

### ESP32-S2 Mini

- Environment в `PlatformIO`: `lolin_s2_mini`
- Дисплей: `SSD1306 128x64` по `I2C`
- Адрес дисплея: `0x3C`
- I2C:
  - `SDA = GPIO18`
  - `SCL = GPIO33`
- UART:
  - `TX = GPIO5`
  - `RX = GPIO3`
- При старте:
  - `GPIO35` устанавливается в `HIGH`
  - `GPIO37` устанавливается в `LOW`
- Линия сброса внешнего устройства: `GPIO37`

## Структура проекта

Заголовочные файлы вынесены в `include/`, реализации лежат в `src/`.

Основные модули:

- `include/define.h` — общие define'ы, выбор платы, флаг экранной/безэкранной сборки
- `include/eeprom.h` — ключи настроек и singleton с инициализацией БД
- `src/app_globals.cpp` — глобальные объекты проекта (`db`, `sett`, `udp`, `display`, `uartQueue`)
- `src/hardware.cpp` — инициализация пинов, `Serial`, `LittleFS`
- `src/display.cpp` — локальный OLED и управление яркостью
- `include/network_bridge.h` — публичный интерфейс сетевого модуля
- `src/network/` — сетевые функции в Android-подобном стиле: один `.cpp` = одна функция
- `src/network/initWiFi.cpp` — подключение Wi-Fi и fallback в AP
- `src/network/initUDP.cpp` — старт UDP на порту `82`
- `src/network/handleExternalScreenUdp.cpp` — обработка framebuffer для внешнего экрана
- `src/network/sendUdpMessage.cpp` — отправка null-terminated строки по UDP
- `src/network/sendUdpMessageLen.cpp` — отправка заданного количества байт
- `src/network/sendUdpBroadcast.cpp` — broadcast-отправка на `255.255.255.255:8888`
- `src/network/sendUdpPacket.cpp` — общая низкоуровневая helper-функция отправки
- `src/network/WifiCurrentPowerString.cpp` — преобразование кода мощности Wi-Fi в читаемую строку
- `src/network/drawStartupVersionFooter.cpp` — helper для стартового экрана с версией прошивки
- `src/settings_portal.cpp` — страница настроек `SettingsGyver`
- `src/uart_bridge.cpp` — чтение UART и пересылка данных в UDP
- `src/setup.cpp` — порядок инициализации
- `src/main.cpp` — основной `loop()`

## Сборка

Установите `PlatformIO`, откройте проект и выберите нужное окружение.

Сборка для `ESP32-C3`:

```bash
platformio run -e esp32-c3-devkitm-1
```

Сборка для `ESP32-S2 Mini`:

```bash
platformio run -e lolin_s2_mini
```

Загрузка прошивки выполняется через то же окружение, которое вы собираете.

## Безэкранная сборка

Для варианта без локального OLED добавьте флаг:

```text
-DPROJECT_NO_SCREEN=1
```

в `build_flags` нужного environment в `platformio.ini`.

Что отключается в таком варианте:

- локальный OLED-дисплей
- управление яркостью экрана
- локальный экран состояния
- режим внешнего экрана по UDP

Остальная логика проекта продолжает работать:

- Wi-Fi
- веб-портал настроек
- UART -> UDP
- `broadcast`
- `echo`
- настройка мощности Wi-Fi

## Настройка

Через веб-интерфейс можно задать:

- IP клиента
- скорость UART
- режим `broadcast`
- режим `echo`
- параметры Wi-Fi
- мощность Wi-Fi
- яркость локального экрана `0..255` при экранной сборке
- использование внешнего экрана по UDP при экранной сборке

Также через портал доступны:

- перезагрузка ESP32
- аппаратный импульс сброса на внешнее устройство
- очистка базы настроек

## Сетевые параметры

- Передача данных по UDP: порт `8888`
- Внешний экран по UDP: порт `82`
- Скорость лога в `Serial`: `460800`

## Примечания

- На некоторых мини-платах Wi-Fi может работать нестабильно при мощности выше `8.5 dBm`. Если появляются проблемы, попробуйте уменьшить мощность через веб-интерфейс.
- Для `ESP32-S2 Mini` дисплей и UART уже разведены на разные пины, чтобы их можно было использовать одновременно.
- Для внешнего экрана ожидается пакет ровно `1024` байта, это размер framebuffer для `SSD1306 128x64`.
- В безэкранной сборке экранный код не просто скрыт в UI, а выключается на этапе компиляции.
